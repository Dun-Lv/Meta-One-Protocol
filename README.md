# 元一协议 | Meta One Protocol
## 📜 序：为什么是“元一”？

世界正面临协作效率低下与意义缺失的危机。旧有的权力结构与分配模式已显现疲态。
“元一协议”试图探索一种新的可能性：一个以**绝对平等**为基础，以**过程积累**为核心资产的社会技术实验。
我们试图回答一个问题：如果一切从头再来，人类应该如何协作？
## ⚖️ 元规则 (The Meta Rules)

任何系统的运行，都必须建立在一套无需证明的公理之上。

**1. 存在第一 (Existence First)**
*   肉体生存权与心理健康权，是系统提供的绝对底线保障。
*   没有个体的存活与健全，任何宏大的协作都毫无意义。

**2. 价值自赋 (Self-valuation)**
*   个体的价值不由他人的评分或系统的指标决定。
*   系统只负责记录与守护这种自我定义的权利，绝不进行任何形式的“价值审判”。

**3. 过程即价值 (Process as Value)**
*   结果固然重要，但**可被自己审计、可被学习的实践过程**，才是系统最核心的资产。
*   失败的尝试，只要过程真实，其价值远高于平庸的成功。
## ⚙️ 核心机制 (Core Mechanisms)

为了实现上述规则，系统设计了以下三大支柱：

*   **生存保障层**
    *   **机制**：无条件基本收入（UBI）或资源平均分配。
    *   **目的**：剥离生存焦虑，让个体回归创造与思考。

*   **发展燃料层**
    *   **机制**：双轨制资金池——“按需分配”（民主共识） + “混沌资金”（保护少数创新）。
    *   **目的**：既保证集体方向的稳健，又保留突破常规的火种。

*   **权力制衡层**
    *   **机制**：一人一票（1人 = 1权）。
    *   **引擎**：动态共识引擎（见下文）。
# ⚖️ 权力制衡

## 根本原则
- **一人一票**：政治权力绝对平等。

## 决策核心：动态共识引擎
**目标**：通过迭代与动态实践，趋近并达成共识。

### 流程
1.  **先实践**
    - 提案可立即由**同意方**在**不损害反对者利益**的范围内执行，并产出实践数据。

2.  **动态投票**
    - 投票状态可自由转换（反对 ⇄ 同意）。
    - 共识状态实时更新与显示。

3.  **保障与优化循环**
    - 若未达100%同意，则进入循环：
        - **同意方**须修改提案或提供利益保障方案。
        - **反对方**可提供建设性意见。
    - 每轮优化后，**全体成员重新判断**，并更新投票状态。

4.  **终止权限**
    - 提案提出者若投反对票，提案**当即永久作废**。

---
*系统在安全实践中迭代，在动态投票中趋近共识。同意方可行动，反对方受保护，提出者握有最终安全阀。*

# ❤️ 心灵支点(The Heart's Fulcrum)

## 作用
- 守护心理健康与意义感，是 **“存在第一”** 在心理层面的基石。
- 为成员提供对抗异化、创伤与意义危机的系统性缓冲。

## 核心机制
- **模块化协议**：可插拔的支持选项（工具/同伴/专业资源）。
- **非侵入、重隐私**：仅成员主动触发，系统不干预、不评判。
- **意义建构**：通过内置认知框架（如五元真理），将个人混沌转化为可理解的“过程知识”。
